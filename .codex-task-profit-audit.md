# 盈利算法全面深度审计

## 审计目标
找出所有可能导致**负 EV 交易被执行**的逻辑漏洞、边界情况、假设错误。

## 核心检查点

### 1. 信号生成层（dex-arbitrage.ts）
- [ ] `calculateEdge()` 计算公式是否正确？单位是否一致（bps vs decimal）？
- [ ] 阈值判断 `edge > threshold` 是否考虑了所有成本？
- [ ] 是否有重复信号风险？同一机会被多次触发？
- [ ] 价格源头的可靠性：如何处理 stale quote？

### 2. 成本模型（cost-model.ts）
- [ ] 双腿 taker fee 计算是否正确？（买入 + 卖出都要付）
- [ ] Slippage 模型是否合理？线性？指数？
- [ ] Gas 成本是否纳入？链上套利必须考虑
- [ ] 是否有隐藏成本未计算？（MEV、失败重试等）

### 3. 模拟器（simulator.ts）
- [ ] `estimate()` 函数是否使用真实成本？
- [ ] pFail（失败概率）是否纳入期望值计算？
- [ ] expectedShortfall（尾部风险）是否考虑？
- [ ] 回测结果是否过度乐观？为什么？

### 4. 执行层（alpha-engine.ts）
- [ ] 信号→交易的转换是否有损耗？
- [ ] 并发处理是否会导致重复执行？
- [ ] 延迟对成交价的影响是否建模？
- [ ] 部分成交如何处理？

### 5. 风控层（risk-engine.ts）
- [ ] 阈值是否足够保守？
- [ ] 是否有最大亏损限制？
- [ ] 连续亏损后的降级机制？

## 交付物
1. 列出所有发现的问题（按严重程度：Critical/Warning/OK）
2. 每个问题给出：文件路径、行号、问题描述、修复建议
3. 估算当前算法的**真实期望收益**（考虑所有成本后）
4. 给出改进建议优先级排序

请用 xhigh 思考模式，逐行审查关键代码，不要跳过任何细节。
